# Global ERP Reference Analysis

클라이언트가 제공한 기존 글로벌 ERP는 조회 전용으로만 확인했다. 데이터 등록, 수정, 삭제, 저장, 전송은 수행하지 않았다. 실제 고객/차량/연락처 데이터는 기록하지 않고, 화면 구조와 입력 항목, 업무 흐름, UI/UX 개선 포인트만 정리한다.

## Observed Menu Structure

- 견적 관리: 견적 리스트, 견적 등록
- 작업 관리: 작업 리스트, 작업 등록
- 청구 관리: 청구 리스트, 다이렉트 청구서 등록
- 재고 관리: 재고 리스트, 재고 등록, 엑셀로 재고 등록
- DB 관리: 부품 마스터 리스트, 차종 DB 관리, 엑셀로 부품 마스터 DB 등록
- DB 관리 (사용안함): 견적 DB 관리, 엑셀로 견적 DB 등록, 청구 DB 관리, 엑셀로 청구 DB 등록
- 업체 관리: 거래처 리스트, 정비소 리스트, 보험사 리스트, 보험사 담당자 리스트
- 내부 커뮤니티: 사내공지, 회의록, 교육 게시판, 자유 게시판
- 통계 관리: 연도별 지점 통계, 전체 지점별/사용자별 통계, 월별 통계

## Common List Screen Pattern

- 좌측 사이드 메뉴
- 상단 업무 건수 배지
- 기간 필터와 빠른 기간 버튼
- 지점/대리점/외주 필터
- 상태 필터
- 검색어 입력
- 결과 테이블
- 행별 관리 버튼
- 등록/수정 화면 공통 검색 팝업: 거래처, 차량번호, 차종, 부품, 정비소, 보험사, 공급처 검색이 반복된다.

## Additional Recheck Notes

2026-05-20 재확인 시에도 데이터 등록, 수정, 삭제, 저장, 전송, 다운로드, 업로드는 수행하지 않았다. 실제 고객/차량/연락처 행 데이터는 기록하지 않았고, 화면 구조와 필드명, 업무 흐름만 확인했다.

- 기존 분석의 핵심 도메인 분류는 유효하다.
- 보강해야 할 핵심 흐름은 `견적 저장`과 `작업 바로 등록`의 분리, `일반 청구`와 `다이렉트 청구`의 분리, `차종 DB`와 `부품 마스터`의 연동이다.
- 보험사/정비소/보험 담당자 데이터는 청구 업무에서 독립 마스터로 취급해야 한다.
- 부품 마스터는 재고와 별개의 기준 데이터이며, 보험청구단가와 공임 산정 정보까지 포함한다.
- 기존 시스템은 오래된 메뉴와 사용안함 메뉴가 남아 있어, 새 ERP에서는 메뉴 노출 정책과 폐기/비활성 상태 관리가 필요하다.

## UX Issues Observed

- 목록 테이블에 표시되는 정보량이 많아 한눈에 스캔하기 어렵다.
- 행별 `수`, `복`, `삭` 같은 축약 버튼은 의미가 불명확하고 오조작 위험이 있다.
- 목록 행 안에서 상태 콤보박스를 바로 변경할 수 있어 실수로 상태가 바뀔 가능성이 있다.
- 견적, 작업, 결제, 보험 청구 정보가 한 화면에 섞여 있어 입력 순서를 이해하기 어렵다.
- 필터가 라디오 버튼 중심이라 지점/대리점 수가 많아질수록 화면이 복잡해진다.
- 차량번호, 차대번호, 연락처 등 민감 정보가 리스트에 직접 노출된다.
- 검색/선택 팝업이 많아 입력 흐름이 끊긴다.
- 캘린더 중심의 작업 일정 확인 흐름이 약하고, 리스트 중심으로 업무를 추적해야 한다.
- 재고는 품번 중심이고 사용자가 직접 관리하는 품목 카테고리 구조가 약해 보인다.
- 저장/삭제/상태변경 같은 위험 작업이 시각적으로 충분히 구분되지 않는다.
- 견적, 작업, 청구, 재고, 마스터 화면에서 유사한 검색 팝업이 반복되어 공통 컴포넌트로 정리할 여지가 크다.
- `사용안함` 메뉴가 실제 메뉴에 남아 있어 운영자가 현재 사용해야 하는 기능과 레거시 기능을 혼동할 수 있다.
- 다이렉트 청구서 등록은 보험/정비공장/부품/공임/세금 정보가 한 화면에 밀집되어 있어 실무자는 익숙해지면 빠르지만 신규 사용자는 진입 장벽이 높다.

## Improvements For Gyeongnam Car Glass ERP

- 첫 화면은 `오늘 작업`, `미수금`, `부족 재고`, `견적 대기` 중심의 대시보드로 구성한다.
- 견적 입력은 `고객/차량 -> 품목/작업 -> 금액 -> 일정 -> 확인` 단계형 흐름으로 나눈다.
- 작업은 리스트와 캘린더를 모두 제공하고, 캘린더에서 일정 상세로 바로 이동한다.
- 행별 액션은 `수정`, `복사`, `삭제`처럼 명확한 텍스트와 아이콘을 함께 사용한다.
- 삭제는 실제 삭제보다 `취소`, `미사용`, `폐기` 상태로 관리한다.
- 상태 변경은 목록에서 즉시 변경하지 않고 상세 화면 또는 확인 모달을 거치게 한다.
- 고객 전화번호, 차대번호, 차량번호는 권한에 따라 마스킹한다.
- 품목 카테고리는 사용자가 직접 생성하고 품목/재고/통계에 연결한다.
- 재고 입고/출고/조정은 모두 이력으로 남기고 작업 완료와 연결한다.
- 견적서 PDF는 확정 시점에 생성하고, 카카오톡에는 만료 토큰이 있는 보기 링크를 보낸다.
- 검색 필터는 라디오 나열 대신 셀렉트, 자동완성, 저장된 필터 프리셋을 사용한다.
- 작업 완료 시 재고 차감과 전표 생성을 자동 또는 확인 후 처리하도록 선택지를 둔다.
- 모든 주요 변경은 감사 로그에 기록한다.
- 견적 저장, 작업 전환, 청구 생성은 한 버튼으로 뭉치지 말고 사용자의 의도를 명확히 구분한다.
- 다이렉트 청구는 일반 청구와 별도 화면 또는 별도 모드로 분리하고, 작업내역 불러오기와 수동 라인 추가를 모두 지원한다.
- 부품 마스터, 차종 DB, 재고를 분리하되 서로 연결된 상태를 명확히 보여준다.
- 보험사 담당자는 현 담당/이전 담당 상태를 관리하고, 청구 시점에는 당시 담당자 이력을 보존한다.
- 엑셀 업로드는 사전 검증, 미리보기, 오류 행 다운로드, 중복 처리 정책을 제공한다.

## Design Impact

기존 시스템의 핵심 도메인은 `견적`, `작업`, `청구`, `재고`, `업체`, `부품/차종 DB`로 확인된다. 경남차유리 ERP의 MVP 설계에 이미 포함된 `견적 -> 작업 -> 재고 -> 전표/결제 -> PDF 발송` 흐름은 기존 시스템을 대체하기에 적절하다.

추가로 개발 초기부터 반영해야 할 항목은 다음과 같다.

- 보험사/정비소/보험 담당자 관리
- 문의 경로 관리
- 방문/출장 구분과 출장자
- 수리구분/수리부위
- 보험청구금액, 보험입금금액, 고객결제금액 구분
- 사진 첨부
- 품번, 품명, 센터가, 평균매입가, 재고위치
- 부품 마스터의 국내/수입, 부품구분, 호환품번, 보험청구단가, M/H, 공임금액
- 차종 DB와 부품 마스터 연동
- 다이렉트 청구서의 정비공장, 보험사 담당자, 접수번호, 기준공임, 공급가/부가세/청구합계 구조
- 보험사 담당자의 현 담당/이전 담당 상태
- 엑셀 업로드/다운로드
