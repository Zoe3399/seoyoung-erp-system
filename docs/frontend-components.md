# 프론트 공통 컴포넌트 정리

프로토타입에서 반복되는 UI는 `apps/web/src/components/common.tsx`로 분리한다.

## 컴포넌트

- `BrandIdentity`: 로고와 상호 표시. 로그인과 상단 헤더에서 같이 사용한다.
- `Panel`: 제목, 우측 버튼, 본문을 가진 기본 영역.
- `RecordToolbar`: 목록 화면의 검색, 필터, 건수, 등록 버튼 영역.
- `SearchInput`: 전체 검색과 자동완성 후보를 가진 검색 입력.
- `FilterTabs`: `전체`, `미수`, `정상` 같은 목록 필터.
- `DetailDrawer`: 항목 클릭 시 오른쪽에서 열리는 상세 패널.
- `ActionFooter`: 저장/닫기 같은 주요 버튼을 하단에 고정 배치하는 버튼 묶음.
- `FormSection`: 입력 폼을 쉬운 제목 단위로 나누는 구역.
- `KpiCard`: 홈/재고 화면의 숫자 요약 카드.
- `DataTable`: 견적, 청구, 재고, 고객 목록 표.
- `StatusPill`: 정상, 미수, 부족 같은 상태 표시.
- `PriorityItem`: 처리 필요 목록 항목.
- `InfoItem`: 고객 상세의 짧은 정보 항목.
- `Typeahead`: 검색형 선택 입력.
- `Field`: 일반 입력.
- `ToolbarButtons`: 검색/등록 버튼 묶음.
- `LineItem`: 청구 항목 한 줄.
- `CalcRow`: 자동 계산 결과 한 줄.
- `HistoryItem`: 고객 상세의 업무 이력 카드.
- `SettingRow`: 설정 화면 항목.

## 정리한 내용

- 같은 브랜드 영역이 상단과 사이드바에 중복 표시되지 않도록 사이드바의 브랜드 블록은 제거했다.
- 로고/상호 표시는 `BrandIdentity` 하나로 관리한다.
- 사이드바는 메뉴 탐색만 담당하게 정리했다.
- 작업 화면은 작업과 일반 할 일을 같은 캘린더에 보여주되, 카드 색과 목록 필터로 구분한다.
- 견적, 청구, 재고, 고객, 거래처, 자료 화면은 같은 `RecordToolbar` 패턴으로 검색/필터/등록 위치를 맞췄다.
- 고객 상세는 중앙 모달보다 레퍼런스에 가까운 오른쪽 `DetailDrawer`로 정리했다.

## 화면 내부 패턴

- `CalendarJob`: 작업 일정과 일반 할 일을 같은 달력 카드 모양으로 표시한다.
- `TodayListItem`: 오른쪽 오늘 할 일 목록에서 작업/일반 항목을 한 줄 카드로 표시한다.
- `WorkListFilter`: `전체`, `작업`, `일반`으로 오늘 목록을 빠르게 좁힌다.
- `목록 화면`: 위에는 검색/필터/등록, 아래에는 표 또는 리스트를 둔다.
- `상세 화면`: 항목을 누르면 오른쪽 상세 패널에서 고객/작업/자료 이력을 확인한다.
- `입력 화면`: 긴 입력은 `FormSection`으로 나누고 저장 버튼은 `ActionFooter`에 모은다.
- `대시보드 바로가기`: 제목과 링크를 입력해 자주 쓰는 외부 사이트를 새 탭으로 연다. 현재는 프로토타입용 임시 상태이며, 저장형 기능은 백엔드 연결 이후 확장한다.
