# 견적 접수

## 프로토타입 목표

클라이언트가 기존 업무 화면과 비교하면서, 견적 단계에서 어떤 정보를 입력하고 어떤 정보는 나중에 채우는지 확인할 수 있게 만든다.

이번 프로토타입에서 가장 중요한 페이지다.

## 클라이언트 답변 반영

- 처음 견적 저장을 고객 연락처, 문의일, 문의 내용 정도로만 간단하게 하는 안은 `N`.
- `첨부자료` 탭의 견적 접수 화면을 참고해야 한다.
- 차량번호, 차대번호, 브랜드/모델/년식, 수리부위, 부품번호, 옵션, 썬팅은 견적 단계에서 입력 가능해야 한다.
- 위 항목들은 필수값은 아니다.
- 수리구분, 보험사/접수번호는 작업 확정 시점에 입력해도 된다.
- 문의 유입 경로는 전화, 문자, 정비소 소개, 거래처.
- 고객, 차량번호, 부품번호는 직접입력 중심.
- 보험사, 정비소, 차종/모델은 검색입력 중심.
- 검색 결과가 없으면 바로 신규 등록 가능해야 한다.
- 정확한 정보가 없어도 임시 입력 후 저장 가능해야 한다.

## 필수 입력 영역

프로토타입에서는 필수 표시를 하되, 저장 버튼을 과하게 막지 않는다.

- 견적일
- 견적 담당자
- 견적 내용

첨부자료 화면 기준으로는 다음 항목도 중요하게 보인다.

- 대리점 또는 지점
- 외주여부
- 문의경로
- 차량번호/소유자
- 고객 연락처
- 견적내용

## 선택 입력 영역

- 고객명 또는 거래처명
- 연락처
- 차량번호
- 차대번호(VIN)
- 브랜드/시리즈
- 모델명
- 년식
- 수리부위
- 부품번호
- 옵션
- 썬팅
- 입고처/매입처
- 매입금액
- 최종견적내용
- 비고

## 수리구분 옵션

클라이언트가 확정한 기본값:

- 교체
- 복원
- 탈부착
- 썬팅
- 기타

기존 안에 있던 `점검`, `ADAS 보정`은 기본 수리구분에서는 제외한다. 필요하면 옵션이나 비고에 표현한다.

## 수리부위 옵션

클라이언트가 제시한 값:

- 전면
- 후면
- 앞문(운)
- 앞문(조)
- 뒷문(운)
- 뒷문(조)
- 앞삼각(운)
- 앞삼각(조)
- QTR(운)
- QTR(조)
- 루프/파노라마
- 기타

답변에 `후면`이 중복되어 있으므로 프로토타입 옵션에는 한 번만 표시한다.

## 여러 부품/여러 부위

한 견적 또는 작업에 수리부위와 부품이 여러 개 들어갈 수 있다.

프로토타입에서는 다음 중 하나로 먼저 보여준다.

- 수리부위 입력을 태그 형태로 여러 개 추가
- 부품 라인 테이블을 두고 `부품 추가` 버튼 제공

부품 라인에는 최소한 다음을 보여준다.

- 부품번호
- 품목명
- 규격
- 수량
- 단가
- 메모

## 썬팅 처리

썬팅은 단독 작업일 수도 있고, 유리 교체에 포함된 작업일 수도 있다.

프로토타입에서는 다음 두 가지가 보이면 된다.

- 작업구분에서 `썬팅` 단독 선택 가능
- 유리 교체 견적 안에서 `썬팅` 입력란 또는 부품 라인 추가 가능

## 결제 정보

첨부자료 화면에 결제 정보가 포함되어 있으므로 견적 접수 화면에서도 미리 보여준다.

- 결제수단: 현금, 카드, 자차, 대물
- 자기부담금: 0, 20, 30, 50, 기타
- 보험청구금액
- 결제금액

실제 확정은 청구/작업 단계에서 하더라도, 견적 화면에서 메모처럼 입력 가능해야 한다.

## 버튼과 클릭 흐름

- `견적 등록`: 등록 패널 열기.
- `임시 저장`: 목록 상단에 임시 견적이 추가된 것처럼 보여주기.
- `견적 저장`: 목록에 새 견적이 추가된 것처럼 보여주기.
- `작업으로 바로 등록`: 작업 등록 패널 열기.
- `부품번호 검색`: 품목 후보 패널 또는 datalist 표시.
- `정비소 검색`: 거래처 후보 표시.
- `검색 결과 없음`: `신규 등록` 미니 버튼 표시.

## 프로토타입 개발 체크리스트

- 견적 목록에서 검색, 필터, 상세 패널이 동작해야 한다.
- 견적 등록 폼에 위 필드가 화면상 배치되어야 한다.
- 필수/선택 입력이 시각적으로 구분되어야 한다.
- 부품/수리부위 다중 입력 느낌을 보여줘야 한다.
- 작업 전환 버튼을 눌렀을 때 작업 등록 화면으로 이어져야 한다.

## 추후 구현 메모

- 현재 백엔드 DTO는 견적 전체 필드를 저장하지 못하므로, 운영 구현 때 확장해야 한다.
- 견적 저장 시 고객/차량/거래처 신규 생성 여부를 확정해야 한다.
- 견적번호 자동 채번 규칙은 운영 단계에서 확정한다.
