# 청구/보험

## 프로토타입 목표

보험 청구, 정비소 청구, 고객 결제금액을 분리해서 관리해야 한다는 요구를 화면에서 명확히 보여준다.

## 클라이언트 답변 반영

- 청구 구분 기본값은 `Y`.
- 보험 청구 업무는 실제로 자주 발생한다.
- 작업 없이 청구서만 바로 작성하는 경우는 없다.
- 보험 입금이 여러 번 나눠 들어오는 경우는 기본적으로 없다.
- 단, 한 사고 건에서 자차보험 A보험사와 대물보험 B보험사가 비율별로 나뉘어 입금되는 경우가 있다.
- 보험사 입금액과 고객 결제액은 반드시 분리해야 한다.
- 기존 청구서 양식은 `첨부자료` 탭의 자동차 부품 대금 청구서를 참고한다.

## 청구 구분

- 일반
- 자차
- 대물(무과실)
- 대물(과실)
- 배상책임
- 정비소 청구

## 청구 목록 필드

- 청구번호
- 작업일
- 고객 또는 거래처
- 차량
- 청구구분
- 보험사
- 보험접수번호
- 보험청구금액
- 보험입금액
- 고객결제금액
- 미수금
- 상태

## 청구 상세 패널

상세 패널에는 금액을 반드시 분리해서 보여준다.

- 보험청구금액
- 보험입금액
- 고객부담금
- 고객결제금액
- 총 입금액
- 미수금

분리 입금 예외 케이스를 보여주기 위해 `입금 구성` 영역을 둔다.

- 자차보험 A: 70%
- 대물보험 B: 30%
- 고객 결제: 자기부담금

이 값은 프로토타입용 예시로 둔다.

## 청구서 출력 미리보기

첨부자료의 자동차 부품 대금 청구서 양식에 맞춰 다음 항목이 보이게 한다.

- 수리차량
- 차량명
- 보험사
- 정비공장명
- 담당자
- 접수번호
- 공급자 정보
- 청구합계금액
- 차주 부담금액
- 보험회사 지급 책임액
- 부품번호
- 부품 및 작업내역
- 작업구분
- 수량
- 부품단가
- 할인율
- M/H
- 부품금액
- 공임금액
- 비고

프로토타입에서는 실제 PDF 생성 대신 미리보기 카드 또는 출력 버튼만 둔다.

## 버튼과 클릭 흐름

- `청구 보기`: 상세 패널 열기.
- `입금 처리`: 입금액 입력 패널 열기.
- `청구서 보기`: 청구서 미리보기 표시.
- `작업 연결`: 관련 작업 상세로 이동 또는 패널 열기.

## 프로토타입 개발 체크리스트

- 보험입금액과 고객결제금액이 같은 칸에 섞이면 안 된다.
- 청구구분 필터가 있어야 한다.
- 정비소 청구와 보험 청구가 구분되어야 한다.
- 청구서 양식 미리보기가 있어야 한다.
- 작업 없이 청구 생성하는 버튼은 강조하지 않는다.

## 추후 구현 메모

- 보험사 리스트는 현재 제공되지 않았으므로 운영 구현 전 별도 수집이 필요하다.
- 청구서 PDF 생성은 출력물 템플릿 작업과 함께 구현한다.
