# 품목/재고

## 프로토타입 목표

기존 품목리스트와 재고리스트가 실제 자료로 들어왔으므로, 프로토타입에서도 부품 마스터와 실재고가 분리되어 보이게 한다.

## 클라이언트 답변 반영

- 재고 수량은 실제로 맞춰 관리해야 한다.
- 재고 위치 관리는 필요하지 않다.
- 최소 재고 알림은 필요하다.
- 기존 부품 목록은 `품목리스트` 탭 참고.
- 기존 재고는 `재고리스트` 탭의 `재고수량` 기준 참고.
- 기존 자료는 전부 이전해야 한다.

## 품목리스트 데이터 요약

`GPT 질문.xlsx` 기준:

- 총 5,030건
- 품목코드 중복 없음
- 품목명 전체 존재
- 규격정보 일부 누락
- 입고단가 대부분 존재
- 출고단가 값은 매우 적음
- 교환단가 대부분 존재

프로토타입에서는 전체 데이터를 넣기보다 대표 샘플 20-50건 정도를 먼저 보여준다.

## 재고리스트 데이터 요약

`GPT 질문.xlsx` 기준:

- 총 320건
- 품목코드 중복 없음
- `재고수량` 값은 일부 누락
- 출고수량과 재고수량에 음수값이 있어 부호 해석이 필요함

프로토타입에서는 데이터 품질 경고를 `시트 전환` 또는 재고 상세에서 표시한다.

## 재고 화면 구성

상단 KPI:

- 총 보유수량
- 부족 재고
- 확인 필요
- 재고 금액
- 청구 기준 금액

목록 필드:

- 부품번호
- 품명
- 규격 또는 호환차종
- 재고수량
- 최소재고
- 입고단가
- 교환단가
- 상태

상태:

- 정상
- 부족
- 확인필요

## 품목 상세 패널

- 품목코드
- 품목명
- 규격정보
- 입고단가
- 출고단가
- 교환단가
- 현재재고
- 최소재고
- 최근 입출고 이력
- 관련 견적/작업

## 입출고 흐름

프로토타입에서는 실제 계산 대신 흐름을 보여준다.

- 입고
- 출고
- 조정
- 작업 완료로 인한 차감

작업 완료 차감은 작업 페이지와 연결된다.

## 최소 재고 알림

최소 재고보다 현재 재고가 낮으면 `부족` 상태로 표시한다.

프로토타입에서는 최소재고 기준을 임의로 넣어 부족 상태 샘플을 만든다.

## 버튼과 클릭 흐름

- `부품 등록`: 등록 패널 열기.
- `입고`: 입고 입력 미니 폼 표시.
- `출고`: 출고 입력 미니 폼 표시.
- `조정`: 관리자 권한 필요 표시.
- `상세 보기`: 품목 상세 패널 열기.

## 프로토타입 개발 체크리스트

- 품목과 재고가 한 화면에 섞여 있더라도 개념상 분리되어 보여야 한다.
- 재고 위치 필드는 우선순위에서 제외한다.
- 최소 재고 알림은 반드시 보인다.
- 작업 완료 시 재고 차감 흐름과 연결되어야 한다.
- 품목 검색은 부품번호, 품명, 규격 기준으로 가능해야 한다.

## 추후 구현 메모

- 엑셀 전체 이관 전 품목코드 매칭과 음수 수량 규칙을 확인해야 한다.
- 운영 구현에서는 수량 직접 덮어쓰기보다 입출고 이력 기반으로 처리한다.
