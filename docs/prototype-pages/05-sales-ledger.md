# 판매/장부

## 프로토타입 목표

클라이언트가 작업 흐름과 별도로 `판매` 흐름도 필요하다고 답변했으므로, 상품 판매와 장부/전표 흐름을 화면에서 분리해서 보여준다.

## 클라이언트 답변 반영

판매 흐름:

1. 전표 생성
2. 재고 차감
3. 입금/미수 확인

전표 생성 시 필요한 값:

- 거래일자
- 거래처
- 품목
- 금액
- 부가세 적용 여부
- 계산서 발행 여부

계산서 발행은 향후 고도화 단계로 언급됐다.

## 판매관리 화면

상품 판매 또는 매입 입고를 빠르게 입력하는 화면으로 구성한다.

목록 필드:

- 번호
- 거래일자
- 구분
- 거래처
- 품목
- 부품번호
- 수량
- 판매금액
- 결제수단
- 재고 반영 후 수량
- 상태

등록 폼 필드:

- 거래일자
- 거래처
- 품목 검색
- 부품번호
- 수량
- 단가
- 공급가
- 부가세
- 합계
- 부가세 적용 여부
- 결제수단
- 메모

## 장부 화면

작업, 청구, 판매가 전표나 입금 확인으로 이어지는 화면이다.

목록 필드:

- 번호
- 작업일 또는 거래일
- 구분
- 업체/거래처
- 차량
- 작업내용 또는 품목
- 청구금액
- 입금액
- 결제금액
- 상태

## 상태

- 판매완료
- 결제대기
- 출고대기
- 입고대기
- 전표대기
- 입금완료
- 미수

## 버튼과 클릭 흐름

- `판매 등록`: 판매 입력 패널 열기.
- `입고 등록`: 입고 입력 패널 열기.
- `장부 전환`: 견적/작업/청구 건을 장부 기록처럼 보여주기.
- `전표 보기`: 전표 미리보기 표시.
- `거래명세서 보기`: 첨부자료의 거래명세서 양식 미리보기 표시.

## 프로토타입 개발 체크리스트

- 작업 흐름과 판매 흐름이 메뉴에서 구분되어야 한다.
- 판매 저장 시 재고가 차감되는 느낌을 보여줘야 한다.
- 부가세 적용 여부가 화면에 있어야 한다.
- 계산서 발행은 버튼 또는 상태만 보여주고 실제 기능은 만들지 않는다.
- 거래명세서 미리보기 버튼이 있어야 한다.

## 추후 구현 메모

- 전표번호 채번, 세금계산서 파일 연결, 회계 연동은 운영 구현에서 다룬다.
