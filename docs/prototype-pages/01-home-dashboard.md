# 홈/대시보드

## 프로토타입 목표

클라이언트가 앱에 들어왔을 때 오늘 처리해야 할 견적, 작업, 청구, 재고, 미수 상태를 한 번에 볼 수 있게 한다.

홈은 설명용 랜딩 페이지가 아니라 실제 업무 시작 화면이어야 한다.

## 클라이언트 답변 반영

- 핵심 업무 흐름은 `견적접수/작업예약 -> 작업완료 -> 재고차감 -> 전표생성 -> 입금/미수확인`.
- 가장 먼저 필요한 업무는 `견적/작업내역 등록`과 `작업리스트 캘린더화`.
- 판매 흐름도 작업 흐름과 별도로 있어야 한다.
- 빠진 업무로 `판매 전표`, `부가세 적용 여부`, `계산서 발행`, `재고 차감`, `입금/미수 확인`이 언급됐다.

## 화면 구성

- 오늘 작업 요약
- 견적 대기
- 작업 예정
- 청구 대기
- 부족 재고
- 미수/입금 확인
- 업무 바로가기
- 최근 등록된 견적/작업/청구

## 프로토타입에서 보여줄 상태

- `견적중`
- `부품확인`
- `확정`
- `작업전환`
- `작업예정`
- `작업완료`
- `청구대기`
- `입금완료`
- `미수`
- `재고부족`

## 클릭 흐름

- `견적 대기` 카드 클릭: 견적 목록으로 이동하거나 견적 상세 패널 열기.
- `오늘 작업` 클릭: 작업 캘린더의 해당 작업 상세 열기.
- `청구 대기` 클릭: 청구 페이지에서 청구대기 필터 적용.
- `부족 재고` 클릭: 재고 페이지에서 부족 필터 적용.
- `업무 바로가기`는 현재처럼 제목과 URL을 추가하고 새 탭 이동을 보여준다.

## 더미 데이터 메모

- 견적 4-7건
- 작업 5-10건
- 청구 3-5건
- 부족 재고 2-3건
- 미수 거래처 2-3건

## 추후 구현 메모

- 홈 숫자는 실제 DB 집계로 교체한다.
- 바로가기는 사용자별 설정 저장으로 확장한다.
- 알림, 권한별 카드 노출은 추후 구현한다.
