# 프로토타입 페이지별 개발 메모

이 폴더는 `GPT 질문.xlsx`에 들어온 클라이언트 답변을 기준으로, 먼저 프로토타입 화면을 만들기 위한 문서다.

현재 목표는 운영 DB/API 완성이 아니라, 클라이언트가 화면을 보고 업무 흐름과 입력 항목을 검토할 수 있게 만드는 것이다. 실제 저장, 엑셀 이관, 권한 검증, 파일 업로드, PDF 생성은 문서 하단의 `추후 구현 메모`로만 둔다.

## 공통 프로토타입 원칙

- 버튼은 실제 저장 대신 화면 안에서 흐름이 이어지는 것처럼 보이게 만든다.
- 필수 저장 검증은 너무 강하게 막지 않는다. 클라이언트가 빈 값 상태도 확인할 수 있어야 한다.
- 검색 필드는 실제 검색 API 없이 `datalist`, 목데이터, 선택 패널로 먼저 구현한다.
- 항목이 없을 때는 바로 신규 등록할 수 있는 버튼 또는 미니 폼을 보여준다.
- 정확하지 않은 정보도 임시 저장 가능한 화면 상태를 둔다.
- 목록, 상세 패널, 등록 패널, 상태 배지는 기존 공통 컴포넌트 패턴을 재사용한다.
- 프로토타입 문구는 개발 설명이 아니라 실제 사용자가 보는 업무 용어로 쓴다.

## 클라이언트 답변 핵심

- 우선순위는 `견적/작업내역 등록`과 `작업리스트 캘린더화`.
- 큰 작업 흐름은 `견적접수/작업예약 -> 작업완료 -> 재고차감 -> 전표생성 -> 입금/미수확인`.
- 별도로 `판매` 흐름도 필요하다. `전표 생성 -> 재고 차감 -> 입금/미수 확인`으로 본다.
- 견적은 단순 문의 저장보다 기존 업무 화면에 가까워야 한다.
- 차량번호, 차대번호, 브랜드/모델/년식, 수리부위, 부품번호, 옵션, 썬팅은 견적 단계에서 입력 가능해야 한다. 단, 필수는 아니다.
- 수리구분, 보험사/접수번호는 작업 확정 시점에 입력해도 된다.
- 한 작업에 여러 수리부위와 여러 부품이 들어갈 수 있다.
- 작업 완료 시 재고는 즉시 차감하는 흐름으로 보여주되, 오입력 수정 가능성을 남긴다.
- 보험사 입금액과 고객 결제액은 반드시 분리해서 보여준다.
- 품목, 재고, 정비소/거래처 데이터는 기존 엑셀 자료를 기준으로 보여준다.
- 삭제 대신 취소, 보류, 미사용 같은 상태 처리 흐름이 필요하다.

## 첨부자료 해석

`GPT 질문.xlsx`의 `첨부자료` 탭에는 프로토타입에 반영할 참고 이미지가 들어 있다.

- 견적 접수 화면 예시: 빨간 박스 영역이 우선 반영해야 할 입력 항목이다.
- 자동차 부품 대금 청구서: 청구/보험 출력물 후보.
- 견적서: 견적 출력물 후보.
- 거래명세서: 판매/장부 출력물 후보.

## 데이터 시트 요약

- 품목리스트: 5,030건. 품목코드, 품목명, 규격정보, 입고단가, 출고단가, 교환단가, 적요.
- 재고리스트: 320건. 품목코드, 품목명, 규격, 전일재고, 입고수량, 출고수량, 재고수량.
- 정비소(거래처) 리스트: 164건. 거래처코드, 거래처명, 대표자명, 전화, 모바일, Fax, 이체정보.

프로토타입에서는 전체 이관보다 대표 샘플을 화면에 배치하고, `시트 전환` 또는 `자료` 화면에서 원본 데이터가 들어왔다는 사실을 보여주는 것이 먼저다.
