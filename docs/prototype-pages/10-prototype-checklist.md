# 프로토타입 작업 체크리스트

## 1순위

- 견적 접수 화면을 첨부자료 기준으로 보강한다.
- 견적 등록 필드를 필수/선택으로 나눈다.
- 차량번호, 차대번호, 브랜드/모델/년식, 수리부위, 부품번호, 옵션, 썬팅을 견적 화면에 둔다.
- 수리구분 옵션을 `교체, 복원, 탈부착, 썬팅, 기타`로 맞춘다.
- 수리부위 옵션을 클라이언트 답변 기준으로 맞춘다.
- 여러 수리부위/여러 부품 입력 느낌을 추가한다.
- 견적 상세에서 작업 등록으로 이어지는 흐름을 확인한다.

## 2순위

- 작업 캘린더에서 일/주/월/년 보기와 작업 리스트를 다듬는다.
- 작업 등록에 최종견적금액을 추가한다.
- 작업 시간은 오전/오후/미정과 10분 단위 시간을 모두 보여준다.
- 작업 완료 패널에서 사용 부품 확인과 재고 즉시 차감 흐름을 보여준다.
- 품번 수정 가능 버튼을 둔다.

## 3순위

- 청구 화면에서 보험입금액과 고객결제액을 분리한다.
- 청구구분을 `일반, 자차, 대물(무과실), 대물(과실), 배상책임, 정비소 청구`로 맞춘다.
- 분리 입금 예외 케이스를 청구 상세에 샘플로 보여준다.
- 청구서 미리보기 버튼을 추가한다.

## 4순위

- 품목/재고 화면에 기존 엑셀 자료가 반영된 듯한 샘플 데이터를 넣는다.
- 품목과 재고 개념을 분리해서 보여준다.
- 최소 재고 알림을 추가한다.
- 재고 위치 관리는 우선 제외한다.
- 재고 상세에 입고/출고/조정 이력 샘플을 둔다.

## 5순위

- 거래처/정비소 화면에 기존 리스트 기반 샘플을 넣는다.
- 고객/차량은 기존 자료가 없다는 전제를 반영해 직접입력/임시저장 흐름을 보여준다.
- 검색 결과 없음 상태에서 신규 등록 버튼을 보여준다.

## 6순위

- 견적서, 청구서, 거래명세서 미리보기 버튼을 만든다.
- 견적 링크 복사 또는 이미지 저장 버튼을 둔다.
- 자료 화면에서 업무별 첨부 연결을 보여준다.

## 7순위

- 판매 흐름을 작업 흐름과 별도로 보여준다.
- 판매 등록 시 전표, 재고 차감, 입금/미수 흐름을 보여준다.
- 부가세 적용 여부와 계산서 발행 예정 상태를 화면에 둔다.

## 8순위

- 설정 화면에 관리자/직원 권한 차이를 보여준다.
- 직원은 재고조정 제한 안내가 보이게 한다.
- 삭제/취소/보류/미사용 정책을 화면에 반영한다.
- 수정 이력 샘플을 추가한다.

## 이번 프로토타입에서 하지 않아도 되는 것

- 실제 DB 저장
- 실제 엑셀 전체 이관
- 실제 파일 업로드
- 실제 PDF 생성
- 카카오톡/문자 발송 연동
- 세금계산서 파일 관리
- 범용 문서함
- 외부 자료 자동 수집함

위 항목은 화면에서 버튼이나 고도화 후보 정도로만 보여준다.
