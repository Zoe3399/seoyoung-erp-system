# 고객/차량/거래처

## 프로토타입 목표

견적과 작업 입력에서 반복적으로 필요한 고객, 차량, 정비소, 거래처, 보험사 검색 흐름을 보여준다.

## 클라이언트 답변 반영

- 기존 고객 리스트는 없다.
- 기존 차량정보 리스트도 없다.
- 보험사 리스트도 없다.
- 정비소 자료는 `정비소(거래처) 리스트` 탭 참고.
- 정비소, 보험사, 차종/모델은 검색입력 방식이 좋다.
- 고객, 차량번호, 부품번호는 직접입력 중심이다.
- 검색 결과가 없으면 바로 신규 등록할 수 있어야 한다.
- 정확한 정보가 없어도 임시 입력 후 나중에 정리할 수 있어야 한다.

## 고객 페이지

목록 필드:

- 고객명
- 연락처
- 차량
- 차량번호
- 차대번호
- 최근 작업
- 미수
- 상태

고객 상세 패널:

- 기본 정보
- 차량 정보
- 견적 이력
- 작업 이력
- 청구/미수
- 자료
- 메모

프로토타입에서는 고객 데이터가 실제로 없다는 전제를 반영해, 샘플 고객은 적게 두고 신규 입력 흐름을 강조한다.

## 차량 입력 흐름

견적 접수에서 차량번호는 직접 입력할 수 있어야 한다.

차량 관련 필드:

- 차량번호
- 차대번호
- 브랜드
- 시리즈
- 모델명
- 년식

차량번호 검색 결과가 없어도 저장 가능한 상태를 보여준다.

## 거래처/정비소 페이지

`GPT 질문.xlsx` 기준 정비소/거래처 리스트:

- 총 164건
- 거래처코드 중복 없음
- 거래처명 중복 없음
- 대표자명, 전화, 모바일, Fax는 일부 누락
- 이체정보는 모두 등록 상태

목록 필드:

- 구분
- 거래처명
- 대표자명 또는 담당자
- 전화
- 모바일
- Fax
- 이체정보
- 미수/미지급
- 자료

구분 필터:

- 전체
- 정비소
- 보험사
- 매입처
- 일반 거래처

## 검색과 신규 등록

프로토타입에서 필요한 패턴:

- 검색 입력
- 후보 목록
- 결과 없음
- 바로 등록 버튼
- 임시 등록 배지

예시 문구:

- `검색 결과가 없습니다. 임시 거래처로 등록할까요?`
- `차량 정보를 나중에 보완할 수 있습니다.`

## 버튼과 클릭 흐름

- `고객 등록`: 고객 등록 패널 열기.
- `거래처 등록`: 거래처 등록 패널 열기.
- `정비소 검색`: 후보 목록 표시.
- `임시 저장`: 상세 정보 없이 목록에 추가된 것처럼 표시.
- `상세 보기`: 360도 상세 패널 열기.

## 프로토타입 개발 체크리스트

- 고객/차량은 기존 자료가 없으므로 직접입력과 임시저장 흐름을 보여준다.
- 거래처/정비소는 기존 리스트가 있다는 느낌을 목데이터로 보여준다.
- 견적 접수 화면에서 정비소 검색 흐름과 연결되어야 한다.
- 상세 화면에서 견적, 작업, 청구, 자료가 연결되어 보이면 좋다.

## 추후 구현 메모

- 운영 구현 전 보험사 리스트를 별도 수집해야 한다.
- 거래처코드는 사업자번호처럼 보이는 값이 있으므로 민감정보 노출 정책을 정해야 한다.
