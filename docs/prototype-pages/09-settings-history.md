# 설정/권한/이력

## 프로토타입 목표

삭제, 취소, 미사용, 권한, 수정 이력에 대한 클라이언트 답변을 화면 정책으로 보여준다.

## 클라이언트 답변 반영

- 견적/작업내역은 삭제, 취소, 보류가 가능해야 한다.
- 거래처/품목 같은 리스트형 항목은 삭제 또는 미사용 처리가 가능해야 한다.
- 기존에 이미 해당 정보로 등록했던 내역에서는 정보가 삭제되지 않아야 한다.
- 사용자는 관리자와 직원 정도로 나누면 된다.
- 직원이 수정하면 안 되는 업무는 재고조정이다.
- 누가 언제 수정했는지 이력을 남겨야 한다.

## 권한 구분

프로토타입에서는 두 권한만 보여준다.

- 관리자
- 직원

관리자:

- 금액 수정
- 청구 확정
- 재고 조정
- 입금 처리
- 취소 처리
- 기준정보 미사용 처리

직원:

- 견적 등록
- 작업 등록
- 작업 상태 변경
- 고객/차량 조회
- 자료 확인

직원 제한:

- 재고 조정 불가

## 삭제/취소/미사용 정책

견적/작업:

- 삭제
- 취소
- 보류

프로토타입에서는 실제 삭제보다 상태 변경으로 보여주는 것이 좋다.

품목/거래처:

- 삭제
- 미사용

이미 연결된 과거 견적/작업/청구에는 당시 이름이 그대로 남아야 한다는 설명을 설정 화면에 둔다.

## 이력 화면

수정 이력 샘플:

- 수정일시
- 수정자
- 대상
- 변경 전
- 변경 후
- 사유

프로토타입에서는 주요 상세 패널에 `수정 이력` 탭을 공통으로 보여준다.

## 설정 페이지 구성

- 사용자 권한
- 자동 계산 기준
- 상태 관리 기준
- 수정 이력
- 삭제/미사용 안내

## 버튼과 클릭 흐름

- `권한 보기`: 권한 상세 카드 표시.
- `재고 조정`: 직원 모드에서는 제한 안내 표시.
- `취소 처리`: 사유 입력 패널 표시.
- `미사용 처리`: 확인 모달 표시.
- `수정 이력`: 이력 패널 표시.

## 프로토타입 개발 체크리스트

- 관리자/직원 권한 차이가 화면에서 보이면 된다.
- 재고 조정 제한은 반드시 보인다.
- 삭제보다 취소/보류/미사용 상태 처리가 강조되어야 한다.
- 상세 패널에 수정 이력 샘플이 있으면 좋다.

## 추후 구현 메모

- 실제 권한 검증은 API와 라우팅 단계에서 구현한다.
- 감사 로그는 운영 DB에 별도 저장한다.
