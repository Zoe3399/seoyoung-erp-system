# 작업 캘린더/작업 등록

## 프로토타입 목표

견적이 확정된 뒤 작업으로 이어지고, 작업 리스트가 캘린더로 보이는 흐름을 클라이언트가 확인할 수 있게 한다.

## 클라이언트 답변 반영

- 견적에서 작업으로 바로 등록하는 흐름은 `Y`.
- 작업 등록 시 `최종견적(금액)`이 추가로 필요하다.
- 작업 시간은 오전/오후만으로도 저장 가능해야 한다.
- 정확한 시간은 10분 단위로 입력 가능해야 한다.
- 작업 사진 첨부는 당장 필요하지 않다. 아직 해당 단계는 구상 중이다.
- 작업 완료 버튼 선택 시 재고는 즉시 차감되는 흐름이 맞다.
- 단, 이후 재고번호 또는 품번 수정이 가능해야 한다. 오입력 가능성이 있다.

## 작업 등록 필드

- 작업일
- 작업시간
- 오전/오후/시간 미정
- 10분 단위 시간
- 방문/출장/탁송
- 출장자 또는 작업 담당자
- 출장 주소
- 작업 메모
- 최종견적금액
- 수리구분
- 보험사
- 접수번호
- 사용 부품

## 캘린더 화면

프로토타입에서는 다음 보기 전환을 제공한다.

- 일
- 주
- 월
- 년

작업 카드에는 다음을 표시한다.

- 시간
- 고객 또는 거래처
- 차량
- 작업내용
- 방문/출장
- 담당자
- 상태

## 작업 상태

- 예정
- 진행중
- 완료
- 보류
- 취소

## 작업 완료 패널

작업 완료 버튼을 누르면 다음 내용을 보여준다.

- 작업 요약
- 사용 부품 확인
- 재고 즉시 차감 예정 안내
- 품번 수정 가능 버튼
- 청구/전표 생성 후보
- 완료 처리 버튼

사진 첨부는 메인 흐름으로 강조하지 않는다. 필요하면 작은 보조 버튼으로만 둔다.

## 재고 차감 표현

프로토타입에서는 실제 계산보다 사용자가 이해할 수 있는 흐름을 우선한다.

예시:

- `86111-AT080 1개 차감 예정`
- `완료 시 재고 3개 -> 2개`
- `품번 수정`
- `재고 차감 완료`

## 버튼과 클릭 흐름

- 견적 상세의 `작업 등록`: 작업 등록 패널 열기.
- 작업 카드 클릭: 작업 상세 패널 열기.
- `작업 완료`: 완료 패널 열기.
- `품번 수정`: 사용 부품 라인을 편집 가능 상태로 변경.
- `보류`: 상태를 보류로 변경한 것처럼 보여주기.
- `취소`: 취소 사유 입력 미니 폼 표시.

## 프로토타입 개발 체크리스트

- 견적 상세에서 작업 등록 패널로 넘어가야 한다.
- 작업 등록 패널에 최종견적금액이 있어야 한다.
- 캘린더에 작업과 일반 할 일이 함께 보여야 한다.
- 오전/오후/미정과 10분 단위 시간이 모두 선택 가능해야 한다.
- 완료 처리 시 재고 차감 흐름이 시각적으로 보여야 한다.

## 추후 구현 메모

- 실제 재고 차감은 운영 구현 때 트랜잭션 이력으로 남긴다.
- 작업 완료 후 전표/청구 자동 생성 여부는 사용자 확인 흐름으로 확정한다.
