# 자료/첨부/출력물

## 프로토타입 목표

견적서, 청구서, 거래명세서 같은 출력물과 업무별 첨부자료가 어떻게 연결되는지 보여준다.

## 클라이언트 답변 반영

- 견적서 PDF 또는 출력 양식이 필요하다.
- 청구서 PDF 또는 출력 양식이 필요하다.
- 현재 사용하는 견적서, 거래명세서 양식이 첨부됐다.
- 청구서 양식도 첨부자료 탭에 있다.
- 고객에게 견적서 링크를 보내거나 이미지를 바로 보낼 수 있으면 좋다.
- 작업 사진 첨부는 당장 필수는 아니다.
- 범용 문서함, 외부 자료 수집함, 세금계산서 파일 관리는 향후 고도화 후보로 필요하다.

## 자료 페이지

목록 필드:

- 파일명
- 연결 업무
- 구분
- 고객/거래처
- 등록일
- 상태

구분:

- 견적
- 청구
- 작업
- 기준정보
- 거래처

## 업무별 첨부

프로토타입에서 보여줄 연결 대상:

- 견적: 파손 사진, 견적 관련 이미지, 견적서
- 작업: 작업 전/후 사진, 사용 부품 확인 이미지
- 청구: 청구서, 보험 관련 자료
- 거래처/정비소: 사업자등록증, 통장사본, 계약서

작업 사진 첨부는 아직 주요 흐름이 아니므로 보조 기능으로만 보여준다.

## 출력물 미리보기

첨부자료 기준으로 다음 미리보기 버튼을 둔다.

- 견적서 보기
- 청구서 보기
- 거래명세서 보기

프로토타입에서는 실제 PDF 생성 대신 이미지형 미리보기 또는 표 형태 미리보기로 충분하다.

## 견적서 출력 항목

- 일자
- 차량정보
- 공급자 정보
- 합계금액
- 품목명/규격
- 수량
- 단가
- 공급가액
- 부가세
- 적요
- 유효기간
- 비고

## 거래명세서 출력 항목

- 일자
- 차량정보
- 공급자 정보
- 합계금액
- 품목명/규격
- 수량
- 단가
- 공급가액
- 부가세
- 적요
- 유효기간
- 비고

견적서와 유사하지만 문서 제목과 용도가 다르다.

- 차량명

- 수량
- 부품단가
- 공임금액

## 링크/이미지 발송 표현

프로토타입에서는 다음 버튼만 먼저 보여준다.

- `견적 링크 복사`
- `이미지로 저장`
- `발송 이력 보기`

실제 카카오톡, 문자 연동은 하지 않는다.

## 버튼과 클릭 흐름

- `자료 등록`: 파일 등록 패널 열기.
- `보기`: 자료 상세 패널 열기.
- `연결 업무 열기`: 연결된 견적/작업/청구 상세로 이동.
- `견적 링크 복사`: 복사 완료 토스트 표시.
- `출력물 보기`: 출력물 미리보기 열기.

## 프로토타입 개발 체크리스트

- 자료 화면은 독립 목록으로 보여준다.
- 견적/청구/거래명세서 미리보기 버튼이 있어야 한다.
- 고객에게 링크 또는 이미지로 보낼 수 있다는 흐름을 보여준다.
- 범용 문서함은 고도화 후보로만 표시한다.

## 추후 구현 메모

- 파일 업로드, 저장소, 권한, 다운로드 이력은 운영 단계에서 구현한다.
- PDF 버전 관리와 링크 만료 정책은 추후 설계한다.
