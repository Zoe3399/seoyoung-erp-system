# 경남차유리 ERP 개발 전 확인 요청

안녕하세요. ERP 개발 전에 아래 방향으로 설계해도 괜찮을지 간단히 확인 부탁드립니다.

어려운 내용은 개발팀에서 정리하겠습니다. 아래 항목은 보시고 **괜찮음 / 수정 필요 / 추가 필요** 정도로만 의견 주셔도 됩니다.

## 1. 전체 방향

아래 흐름으로 만들려고 합니다.

```text
견적 접수
-> 작업 예약
-> 작업 완료
-> 재고 차감
-> 청구/전표 생성
-> 입금/미수 확인
```

확인 부탁드립니다.

1. 위 흐름으로 업무를 처리하면 괜찮을까요?

2. 위 흐름에서 빠진 업무가 있을까요?

3. 지금 가장 먼저 해결됐으면 하는 업무가 있다면 무엇인가요?

## 2. 견적 접수

견적은 처음부터 모든 정보를 필수로 받지 않고, 간단히 저장할 수 있게 만들려고 합니다.

처음 저장할 때 필요한 값:

- 고객 연락처
- 문의일
- 문의 내용 또는 메모

나중에 채워도 되는 값:

- 고객명
- 차량번호
- 차대번호
- 브랜드/모델/년식
- 수리구분
- 수리부위
- 부품번호
- 옵션
- 썬팅
- 보험사/접수번호

확인 부탁드립니다.

1. 처음 견적 저장은 이렇게 간단하게 해도 괜찮을까요?

2. 처음부터 꼭 받아야 하는 정보가 더 있을까요?

3. 고객 문의는 주로 어디로 들어오나요?
   - 예: 전화, 카카오톡, 문자, 방문, 정비소 소개, 보험사

## 3. 수리구분과 수리부위

수리구분은 아래처럼 시작하려고 합니다.

- 교체
- 복원
- 탈부착
- 점검
- 썬팅
- ADAS 보정
- 기타

수리부위는 아래처럼 시작하려고 합니다.

- 전면유리
- 후면유리
- 운전석 도어유리
- 조수석 도어유리
- 운전석 뒤 도어유리
- 조수석 뒤 도어유리
- 쿼터글라스
- 벤트글라스
- 썬루프/파노라마
- 몰딩
- 카울
- 기타

확인 부탁드립니다.

1. 위 항목으로 시작해도 괜찮을까요?

2. 빼야 할 항목이나 추가해야 할 항목이 있을까요?

3. 한 작업에 수리부위나 부품이 여러 개 들어가는 경우가 있나요?

4. 썬팅은 별도 작업으로 보는 게 좋을까요, 유리 작업의 옵션으로 보는 게 좋을까요?

## 4. 검색 방식

목록이 많아질 수 있는 항목은 직접 전부 보여주지 않고 검색해서 선택하는 방식으로 만들려고 합니다.

예:

- 고객 검색
- 차량번호 검색
- 보험사 검색
- 정비소 검색
- 부품번호 검색
- 차종/모델 검색

확인 부탁드립니다.

1. 위처럼 검색해서 선택하는 방식으로 만들면 괜찮을까요?

2. 검색 결과가 없으면 바로 새로 등록할 수 있게 하면 좋을까요?

3. 정확한 정보가 없을 때 임시로 입력하고 나중에 정리할 수 있게 하면 좋을까요?

## 5. 작업 예약과 완료

견적이 확정되면 작업으로 바로 등록할 수 있게 만들려고 합니다.

작업 등록 시 확인할 값:

- 작업일
- 작업시간 또는 시간 미정
- 방문/출장
- 작업 담당자
- 출장 주소
- 작업 메모

작업 완료 시:

- 작업 사진 첨부
- 사용 부품 확인
- 재고 차감
- 청구 또는 전표 생성

확인 부탁드립니다.

1. 견적에서 작업으로 바로 등록하는 흐름이 괜찮을까요?

2. 작업 시간은 정확한 시간으로 잡나요, 아니면 오전/오후/미정도 필요할까요?

3. 작업 사진 첨부가 필요할까요?

4. 작업 완료 후 재고는 바로 차감할까요, 사용 부품 확인 후 차감할까요?

## 6. 보험과 청구

보험/청구는 아래 구분으로 시작하려고 합니다.

- 일반
- 자차
- 대물(무과실)
- 대물(과실)
- 배상책임
- 정비소 청구

청구에서는 보험사 입금액과 고객 결제액을 따로 확인할 수 있게 만들려고 합니다.

확인 부탁드립니다.

1. 위 청구 구분으로 시작해도 괜찮을까요?

2. 보험 청구 업무가 실제로 자주 발생하나요?

3. 작업 없이 청구서만 바로 작성하는 경우도 있나요?

4. 보험 입금이 여러 번 나눠 들어오는 경우가 있나요?

5. 현재 사용 중인 청구서 양식이 있다면 공유 가능할까요?

## 7. 재고 관리

재고는 부품 목록과 실제 재고 수량을 나누어 관리하려고 합니다.

예:

- 부품번호/품명/호환품번/센터가
- 현재 재고
- 재고 위치
- 매입처
- 입고/출고/조정 이력
- 최소 재고 알림

확인 부탁드립니다.

1. 재고 수량을 실제로 맞춰 관리해야 하나요?

2. 재고 위치 관리가 필요할까요?

3. 최소 재고 알림이 있으면 좋을까요?

4. 기존 부품 목록이나 재고 엑셀 자료가 있다면 공유 가능할까요?

## 8. 견적서/청구서/기존 자료

확인 부탁드립니다.

1. 견적서 PDF 또는 출력 양식이 필요할까요?

2. 청구서 PDF 또는 출력 양식이 필요할까요?

3. 현재 사용하는 견적서, 청구서, 작업지시서 양식이 있다면 공유 가능할까요?

4. 기존 고객, 차량, 부품, 재고, 보험사, 정비소 자료가 있다면 공유 가능할까요?

5. 기존 자료는 전부 이전해야 할까요, 아니면 필요한 기준정보만 먼저 옮기면 될까요?

## 9. 권한과 이력

삭제는 실제 삭제하지 않고 취소/미사용 상태로 남기고, 주요 수정 이력은 기록하려고 합니다.

확인 부탁드립니다.

1. 삭제 대신 취소/미사용 처리로 남기는 방식이 괜찮을까요?

2. 사용자는 관리자와 직원 정도로 나누면 될까요?

3. 직원이 수정하면 안 되는 업무가 있을까요?
   - 예: 금액 수정, 청구 확정, 재고 조정, 입금 처리, 취소 처리

4. 누가 언제 수정했는지 이력을 남기면 괜찮을까요?
