## 1. 문서 목적

본 문서는 경남차유리 업무관리 ERP에서 필요한 화면, 데이터, 계산 규칙, 문서 출력, 정산 흐름을 하나의 요구사항으로 정리하기 위한 문서이다.

현재 확인된 핵심 목표는 다음과 같다.

1. 기존 실무자가 엑셀/HTML 시트로 관리하던 작업, 단가, 결제, 매입, 회계, 보증서, 외부 사이트 정보를 ERP 화면으로 이전한다.
2. 차량번호, 차종, 작업일, 거래처, 작업내역을 중심으로 견적, 작업, 보증서, 결제, 정산 데이터를 연결한다.
3. 관리자가 직접 입력하던 반복 결제, 카드 입금, 보험 청구 계산, 필름 재단 정보 확인 업무를 화면에서 빠르게 처리할 수 있게 한다.
4. 우선 프론트 퍼블리싱 프로토타입에서는 실제 DB 없이 샘플 데이터 기반으로 화면 구조, 검색, 미리보기, PDF 보기 흐름을 검증한다.

### 18.6 UI/UX 공통 요구사항

참고 자료: Flextudio OMS 글에서 주문관리시스템은 주문 접수, 재고 확인, 출고, 거래명세서, 결제 상태를 한 번에 관리하는 흐름으로 설명된다. 본 ERP의 판매관리도 같은 관점으로 상품 판매, 매입 입고, 재고 차감, 결제 확인을 하나의 운영 흐름으로 설계한다.

| 항목 | 요구사항 |
| --- | --- |
| 플로우 중심 | 페이지를 과하게 나누지 않고 견적, 작업, 판매, 결제 상세에서 다음 업무 단계로 자연스럽게 전환한다. |
| 목록/상세 역할 | 목록은 검색과 필터, 상세는 업무 처리와 다음 액션 중심으로 구성한다. |
| 기준정보 노출 | 차량, 품목, 거래처, 단가는 업무 화면에서 검색/추천으로 쓰고, 마스터 화면에서는 전체 목록을 관리한다. |
| 후속 액션 | 작업 상세에서 재고 차감, 보증서 발급, 청구/전표 생성, 결제 확인을 바로 시작할 수 있어야 한다. |
| 입력 UI | 신규 등록이나 상세 입력 폼은 기본 목록 화면에 상시 노출하지 않고 버튼 클릭 시 모달 또는 드로어로 제공한다. |
| 목록 화면 | 목록 화면은 보기, 검색, 필터, 상태 확인, 다음 처리 액션에 집중하고 큰 입력 폼을 상단에 고정 노출하지 않는다. |
| 입력 유형 | 짧은 신규 등록은 모달, 긴 등록/상세 수정은 큰 모달 또는 우측 드로어, 완료/확인은 확인 모달, 출력/도장 조정은 미리보기 모달로 제공한다. |
| 정보 우선순위 | 운영 화면은 입력 폼보다 목록, 상태, 미처리 건, 알림을 먼저 보여준다. |
| 밀도 | 관리자 화면은 카드형 소개 화면이 아니라 반복 업무에 맞는 밀도 높은 테이블과 필터를 우선한다. |
| 화면 흐름 | 상단 KPI, 중간 업무 리스트, 우측/하단 보조 패널, 상세/등록 드로어 순서로 배치한다. |
| 상태 표현 | 작업, 판매, 출고, 결제, 보증서, 계산서 상태는 색상 배지와 짧은 문구로 일관되게 표현한다. |
| 빠른 검색 | 차량번호, 고객명, 거래처, 품번, 차종, 작업번호를 한 검색창에서 찾을 수 있어야 한다. |
| 직접 입력 최소화 | 차종, 품목, 거래처, 단가, 필름 재단은 기존 마스터에서 자동 추천하고 수기 입력은 예외 처리로 둔다. |
| 상세 진입 | 행 클릭 시 상세 패널 또는 드로어가 열리고, 목록 화면을 벗어나지 않고 상태를 수정할 수 있어야 한다. |
| 모바일/태블릿 | 현장 작업자 화면은 모바일에서도 핵심 상태와 다음 액션이 먼저 보이도록 구성한다. |
| 빈 상태 | 데이터가 없을 때는 등록, 가져오기, 필터 초기화 등 다음 행동을 바로 제공한다. |
| 민감정보 | 외부 계정, 금액 변경, 정산 정보는 권한별로 노출 범위를 다르게 한다. |

### 18.7 판매관리 화면 리디자인 요구사항

현재 프로토타입의 판매관리 화면은 등록 폼이 상단의 큰 영역을 차지해 실제 관리자 운영 흐름과 맞지 않는다. 판매관리는 OMS 관점으로 주문, 재고, 출고/입고, 결제 상태를 먼저 파악하고 필요한 건만 빠르게 처리하는 화면이어야 한다.

| 영역 | 개선 요구사항 |
| --- | --- |
| 상단 KPI | 상품 판매액, 출고 대기, 결제 대기, 입고 대기, 재고 부족을 표시한다. |
| 핵심 목록 | 판매/매입 내역 테이블을 화면 중심에 배치한다. |
| 목록 컬럼 | 주문번호, 접수일/거래처, 품목/품번, 주문수량/재고, 판매금액/마진, 결제상태, 출고/입고상태, 처리 버튼 |
| 필터 | 전체, 상품판매, 매입입고, 결제대기, 출고대기 필터를 제공한다. |
| 검색 | 품목, 품번, 고객/업체, 결제수단, 상태를 검색한다. |
| 처리 흐름 | 주문접수, 재고확인, 출고/인수, 결제확인 단계의 미처리 건수를 보조 패널로 보여준다. |
| 재고 알림 | 부족, 확인필요, 최소수량 이하 품목을 별도 알림으로 보여준다. |
| 등록 폼 | 판매/매입 등록 폼은 상단 주 화면이 아니라 하단 빠른 등록 또는 드로어로 제공한다. |
| 상태 처리 | 목록에서 바로 처리, 상세, 결제확인, 출고처리 액션을 수행할 수 있어야 한다. |
| 재고 연동 | 판매 저장 시 재고 자동 차감, 매입 입고 완료 시 재고 증가 흐름을 명확히 보여준다. |
| 정산 연결 | 카드/계좌/외상 결제는 카드매출, 미결 리스트, 대금결제와 연결되어야 한다. |
| 상품 용도 | 상품은 작업용, 판매용, 공용으로 구분한다. 작업용은 작업원가/재고차감에, 판매용은 상품매출/정산에, 공용은 양쪽에 연결한다. |
| 엑셀 일괄등록 | 상품 마스터는 엑셀 양식 다운로드, 엑셀 붙여넣기 입력, 파일 업로드, 컬럼 검증, 오류 행 표시를 지원한다. |

## 20. 검수 기준

프로토타입 기준 검수:

1. 사이드바에서 4단계 메뉴 구조가 깨지지 않고 표시된다.
2. 보증서 등록 후 입력값이 미리보기 양식에 반영된다.
3. 보증서 미리보기는 A4 비율로 보이고 브라우저 인쇄/PDF 저장이 가능하다.
4. 단가표에서 차종/브랜드/품목 검색이 가능하다.
5. 차종을 선택하거나 검색하면 연결 가능한 필름 재단 정보가 표시된다.
6. 대금결제 등록 화면에서 반복 주기와 종료일을 입력할 수 있다.
7. 카드매출 화면에서 입금금액 입력 시 수수료와 수수료율이 계산된다.
8. 매입 화면에서 기준월별 합계와 결제 상태를 확인할 수 있다.
9. 미결 리스트에서 거래처별 미결 금액과 후속 메모를 확인할 수 있다.
10. 외부 계정정보는 원문 비밀번호가 화면에 그대로 노출되지 않는다.

실제 개발 기준 검수:

1. 주요 금액 변경은 변경 이력에 남는다.
2. 보증서, 결제, 매입, 카드매출은 작업 또는 거래처와 연결된다.
3. 기존 BOB0 자료를 가져올 때 오류 행을 확인하고 제외/수정할 수 있다.
4. 보험 청구 계산 결과에 적용 산식과 근거 금액이 함께 저장된다.
5. 회계 계정코드 추천은 관리자가 수정할 수 있고 수정 결과를 다음 분류에 재사용할 수 있다.
